# Tokenized Stocks

Stockpackz uses **real Robinhood Chain stock tokens** — standard ERC-20 contracts (with ERC-8056 extensions) that represent tokenized exposure to underlying equities and ETFs.

## What is a Robinhood Stock Token?

From [Robinhood Chain documentation](https://docs.robinhood.com/chain/building-with-stock-tokens/):

> Stock tokens are standard ERC-20 tokens… Every stock token has a per-asset Chainlink price feed… Stock tokens also implement ERC-8056 (Scaled UI Amount Extension), which defines the corporate-action multiplier (`uiMultiplier()`).

Key properties:

| Property | Value |
|----------|-------|
| Chain | Robinhood Chain (chain ID **4663**) |
| Decimals | **18** |
| Standard | ERC-20 + ERC-8056 |
| Pricing | Chainlink `AggregatorV3Interface` per token |
| Settlement | On-chain transfer via standard `transfer()` |
| Naming | `{TICKER}` symbol, e.g. `NVDA`, `AAPL`, `SPCX` |

## Stock Logos

Logos are loaded from the **Financial Modeling Prep** equity logo CDN:

```
https://financialmodelingprep.com/image-stock/{TICKER}.png
```

Implementation: `src/components/capsules/stock-logo.tsx`

- White background container for consistent display on dark UI
- Automatic fallback to ticker abbreviation if image fails
- Supports explicit `logoUrl` override per token
- Configured in `next.config.ts` under `images.remotePatterns`

### Why FMP instead of Clearbit?

Clearbit Logo API is deprecated/unreliable for many tickers. FMP provides consistent PNG logos for US equities and is widely used in fintech UIs.

## Pricing

### Preview (current build)

Prices in `tokenized-stocks.ts` (`pricePerShareUsd`) are **static mock values** for UI preview. They approximate real market prices but are not live.

### Production

Read prices from each token's **Chainlink price feed** on Robinhood Chain:

```solidity
AggregatorV3Interface feed = AggregatorV3Interface(NVDA_PRICE_FEED);
(, int256 price, , uint256 updatedAt, ) = feed.latestRoundData();
require(price > 0 && updatedAt > 0, "Invalid price");
```

See [Oracles & Price Feeds](https://docs.robinhood.com/chain/oracles-and-price-feeds/) for feed addresses.

## Fractional Amounts

Pack pulls grant **fractional token amounts**, matching how Robinhood Chain handles fractional shares natively (e.g. `0.01 TSLA`).

Amount ranges by rarity (game layer):

| Rarity | Token Amount Range |
|--------|-------------------|
| Common | 0.01 – 0.05 |
| Rare | 0.03 – 0.08 |
| Epic | 0.05 – 0.12 |
| Legendary | 0.08 – 0.20 |

USD value = `tokenAmount × pricePerShareUsd` (from Chainlink in production).

Generated by `generatePullAmount()` in `src/lib/tokenized-stocks.ts`.

## ERC-8056 Corporate Actions

Stock tokens use `uiMultiplier()` for dividends and splits without rebasing balances:

```
underlying shares = raw token amount × uiMultiplier ÷ 1e18
```

Use `balanceOfUI(account)` for display-friendly underlying share counts.

Events to monitor:
- `UIMultiplierUpdated` — corporate action applied
- `TransferWithScaledUI` — transfer with both raw and UI-adjusted values

## Adding a New Token

1. Verify contract on [Robinhood Chain Token Contracts](https://docs.robinhood.com/chain/contracts/)
2. Add entry to `TOKENIZED_STOCKS` in `src/lib/tokenized-stocks.ts`
3. Set `logoUrl: getStockLogoUrl("TICKER")`
4. Add to relevant capsule in `src/lib/mock-data.ts`
5. Update this documentation
